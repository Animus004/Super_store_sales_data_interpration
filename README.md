# Superstore Sales Dashboard (Case Study)



## 1. Project Overview
In this project, I worked on Superstore sales data to study sales and performance patterns across the dataset. I used Excel for analysis and built dashboard/report outputs for business review.

## 2. Business Problem
Commercial teams need clear sales visibility to answer:
- Which segments and categories perform better?
- Where are potential performance gaps?
- What actions should be prioritized based on the analysis?

## 3. Dataset
- **Source file:** `Data/raw/Sample - Superstore.xls`
- **Type:** Retail sales dataset

## 4. Tools & Stack
- Microsoft Excel (data analysis and dashboarding)
- Pivot-based analysis
- Stakeholder report documentation

## 5. Analytical Approach
1. Prepared the source file for analysis.
2. Created pivot-based and summary analysis views.
3. Built dashboard visuals for performance tracking.
4. Documented findings and stakeholder questions.

## 11. Business Recommendations

- Investigate root causes behind high-sales but low-profit categories before scaling investment.
- Prioritize campaigns for high-profit customer segments.
- Review underperforming states with strong sales to identify operational issues.
- Reassess repeated-loss table products for discontinuation or pricing strategy changes.
- Plan promotions and inventory around strong seasonal months.

## 12. Limitations

- Some stakeholder questions are still listed for further analysis and are not fully resolved in the current version.
- Additional investigation is required to confirm operational causes behind low-profit states.
- The project is based on the available historical dataset and report scope.
